Pixmicat!2 (暫定) : New Pixmicat! Frame

  • 全新開發撰寫，藉以擺脫掉 GazouBBS / futaba 包裹，但實質上已是全新開發的版本
    (硬要說一樣也只有排版...其他措施貼圖板都這個樣，能說是抄襲嗎？)
  • 框架核心由以往的巨核模式(所有功能都在同一檔案)變為微核模式(主程式極小，元件極多)
  • 核心函式封裝，讓其他擴充模組可共享使用
  • 從原本的 PHP 小程式變成 Web Application 的概念
  • 遵從 REST (Representational State Transfer) 架構風格，以 pixmicat.php/版面/動作/參數 的 URL 為代表
    (可使用 .htaccess 變成 /版面/動作/參數 比較好看)
  • 本體直接支援多設定檔、多版面設定，同時將各個架構以資料夾分明便於分辨及維護
  • 記錄檔繼承自 PIO 版本結構，也就是說 Pixmicat!2 是以 PIO 版為基礎
  • 完整的頁面快取，只要記錄檔資料沒有更新，就盡量採用快取
  • 原生支援 AJAX，當使用者關閉 JavaScript 也能提供傳統的頁面操作
  • Issues:
    • PIO SQL based (不再支援純文字檔案格式)
    • 各函式封裝、重構 (可重複呼叫, 工作量小): Ref#1973, #2252, #2491
    • PMS 模組掛載點重新架構問題: Ref#1973, #2012, #2331, #2332, #2493
    • PMS 模組設定項目儲存機制，或可擴充為 PMC 的設定項目儲存? (一些暫存資訊如總資料筆數)
    • 可開放API供外部程式呼叫運用?
    • FileIO 計算版面附加圖檔數目及大小的更佳方法 (建立檔案對應表格來取代逐個尋找?)
